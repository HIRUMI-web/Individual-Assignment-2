Brief Project Description

The Student Profile Management System is a dynamic, web-based CRUD (Create, Read, Update, Delete) application designed to streamline the administration of student information. Built using a Java Web architecture (Servlets, JSPs, and JavaBeans), the system securely captures individual student details—such as ID, name, academic program, contact information, personal hobbies, and a brief biography.

To deliver an exceptional user experience, the system replaces traditional, rigid administrative layouts with a highly modern, responsive Cyberpunk/Neon Aesthetic. It features custom HTML5 canvas background animations (falling streams and fluid paint splatters), interactive state-driven forms, and an advanced dynamic rainbow control system that responds fluidly to user hover actions.

List of Implemented Features
Dynamic Student Registration Form: An intuitive input terminal designed with floating glassmorphism text fields, custom animated rainbow placeholders, and structured data validation for tracking core student profiling details.

Centralized Profile Directory (Real-time View): A clean dashboard layout mapping out all registered students onto distinct profile cards, providing administrative overviews at a glance.

Multi-Criteria Search Engine: Real-time lookup mechanics allowing administrators to instantaneously query the student database by entering either a Student ID or Full Name.

Academic Program Filtering: An integrated program filtering mechanism that subsets the global student list, making it easy to isolate and review specific cohorts (e.g., Bachelor of Computer Science).

Interactive Control State Transitions: Highly animated action elements, including:

Apply Filter Button: Transitions smoothly from a permanent rainbow gradient background to a solid black body with isolated text/border rainbow glow patterns upon hover.

Reset & Back Navigation Links: Synchronized interactive boundaries that instantly cycle between solid black backdrops and full rainbow spectrum fills when hovered over.

Full Profile Modification Workflow (CRUD): * Deep-linked Edit Form Action parameters to change student data fields in place.

Contextual Delete Actions coupled with defensive JavaScript confirmation modals to prevent accidental profile purging.

Interactive Canvas Background Renderers: * Registration Stage: Running matrix-style linear neon streams animating continuously over a dark layout.

Directory View Stage: Floating organic paint splatter blots running alpha-blended color configurations that pulse in the background without obstructing table readability.
